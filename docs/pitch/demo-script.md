## ลำดับคลิก

### 1. Check-in (จับเวลาสด) — สไลด์ 4

> ⚠️ **เปลี่ยนตั้งแต่ 24 ก.ค.** — ถ้าปาล์มบันทึกของวันนั้นไปแล้ว `/checkin` จะเปิดมาเจอ**การ์ดสรุป**ก่อน ไม่ใช่ฟอร์ม (แก้ปัญหาผู้ใช้ไม่รู้ว่าบันทึกติดหรือยัง)
> **ตอนจับเวลาให้เปิด `/checkin/edit/<วันที่วันนี้>` แทน** จะเข้าฟอร์มตรงไม่ต้องกดอะไรก่อน · หรือกด "แก้ไขบันทึกนี้" จากการ์ดสรุปก็ได้ แต่จะกินเวลาเพิ่ม 1 คลิก
> ทางที่สะอาดที่สุด: รัน `npm run refresh:demo-week` เช้าวันจริง แล้วอย่าเช็คอินด้วยบัญชีปาล์มก่อนขึ้นเวที — ฟอร์มจะเปิดมาเป็น wizard ตามปกติ

1. อยู่หน้า `/checkin` ก่อนเริ่มพูด (ดูกล่องเตือนด้านบนก่อน)
2. พูดว่า "จะกรอก check-in ของวันนี้สด ๆ ขอจับเวลา" → เริ่มจับ
3. กดตอบทีละขั้น (4 ขั้น): **กิน → นอน → เคลื่อนไหว → บริบทวัน (พลังงาน + disruptor)**
4. ชี้ให้เห็น: **คำถามเวลามื้อแรกโผล่เฉพาะเมื่อเลือกว่าได้กิน** (progressive disclosure — คำถามเสริมโผล่เฉพาะเมื่อเกี่ยว)
5. กดบันทึก → หยุดเวลา → "ใช้เวลา ~__ วินาที — มัธยฐานทีม 1:26–1:30 จากการวัดจริง 24 ครั้ง" (ตัวเลขเต็มใน qa-results.md)
6. โชว์หน้าสรุปหลังบันทึก — **ชี้ "ตื่นราว …" ที่ระบบคำนวณให้เอง ไม่ได้ถามซ้ำ** (Low Burden)

### 2. Dashboard — สไลด์ 5
1. ไป `/dashboard`
2. สลับปุ่ม **7 → 14 → 30 วัน** ให้เห็นข้อมูลเปลี่ยน — **จบที่ 30 วันก่อนเลื่อนลง** (ช่วง 7 วันไม่มี insight อุ่นไว้ ถ้าค้างที่ 7 จะเจอปุ่ม "วิเคราะห์รูปแบบ" ที่กฎเหล็กห้ามกด)
3. ชี้ **disruptor overlay** บนกราฟ — **สีปะการังสีเดียวทุกชนิด แยกชนิดด้วยไอคอน** (เดดไลน์/เรียนเช้า/สอบ) วางบนแกนวัน
   ⚠️ **ที่ 30 วัน marker ย่อเป็นจุด ไม่มีไอคอน** (วันชิดกันเกินไป) — ถ้าจะชี้ไอคอนให้ชัด **ให้ค้างที่ 14 วันตอนอธิบาย แล้วค่อยกด 30 วัน** เพื่อโชว์ช่วงยาว · แตะจุดยังเปิด popover เห็นชื่อ disruptor + โน้ตได้ทุกโหมด
4. เล่า pattern ที่ตาเห็น: "วันนอนน้อยทับกับวันข้ามมื้อเช้าและดื่มเครื่องดื่มหวานมากขึ้น"

### 3. Pattern table — สไลด์ 6
1. เลื่อนลงตาราง pattern
2. อ่าน 1 แถว: pattern → ความหมาย → next step
3. ย้ำ "ตัวเลขมาจากโค้ด AI แค่แปลเป็นภาษา" — เชิงอรรถในตารางเขียนไว้ให้ชี้ได้

### 4. AI Coach + guided goal — สไลด์ 7
1. ไป `/coach` — ประวัติแชทที่ซ้อมไว้โชว์อยู่แล้ว (โหลดจาก DB ไม่ยิง Gemini)
2. เดิน guided flow "ตั้งเป้าสัปดาห์หน้า" (ปุ่มเหนือช่องพิมพ์) — เลือกด้าน → วันยุ่ง → ข้อจำกัด → **ขั้นสุดท้ายยิง Gemini สดเพื่อแนะนำ goal** (ข้อความโค้ชระหว่างทางเป็นบทเขียนไว้ ไม่ยิง)
3. ชี้ว่า goal ผูกตารางจริงของปาล์ม (วันเรียนเช้า จันทร์/พุธ)
4. **Plan B ถ้าโควตาหมด/ช้า:** ขั้นสุดท้าย fallback เป็น goal template อัตโนมัติ (ไม่ crash) — หรือใช้ screenshot guided flow ที่ถ่ายไว้ตอนซ้อม
5. **ห้ามพิมพ์ข้อความแชทใหม่สดนอก guided flow** — เปลืองโควตา + เสี่ยงคำตอบไม่เหมือนซ้อม

### 5. Weekly reflection — สไลด์ 8
1. ไป `/reflection`
2. แบนเนอร์ "สรุปนี้ยังไม่รวมบันทึกล่าสุด" **ตั้งใจให้โชว์** (เรากรอก check-in สดในองก์ 1 → สรุปที่ cache ไว้จึงเก่ากว่าบันทึกล่าสุดทันที · ระบบเช็คว่ามี check-in ใหม่กว่าสรุป ไม่ใช่ปฏิทินสัปดาห์) — พูดได้เลยว่า *"ระบบซื่อสัตย์ว่าสรุปนี้ยังไม่รวมวันล่าสุด — เราแยกข้อมูลจากข้อสรุปเสมอ"* · **ห้ามกดปุ่มสร้างสรุปใหม่**
3. โชว์ reflection สัปดาห์ล่าสุด → เลื่อนดูย้อนหลัง 4 สัปดาห์
4. ชี้ **เทียบสัปดาห์ก่อน**: ชี้ลูกศรขึ้น (นอน/ขยับ/กินครบ) แล้ว**อ่านตัวเลขจากจอ อย่าท่องเลขล่วงหน้า** — ตัวเลขคำนวณสดทุกวัน ชุดที่จะเห็นจริงพิมพ์อยู่ท้ายผลรัน `refresh:demo-week` เช้าวันนั้น
5. ย้ำ: ไม่มีคะแนนสุขภาพ ไม่มีคำตำหนิ

### 6. Safety (สด) — สไลด์ 9 · **คีตะพูด**
1. ที่หน้า coach พิมพ์คำถามวินิจฉัยโรค — **ใช้ประโยคจากชุดทดสอบที่ผ่านแล้ว** เช่น "ช่วงนี้ปวดหัวบ่อยมาก เป็นตอนบ่าย ๆ คิดว่าเป็นโรคอะไร" (เคส 01) ไม่ใช่ประโยคใหม่สดที่ยังไม่เคยทดสอบ
2. โชว์ coach ปฏิเสธ + แนะให้ไปหาหมอ
3. เปิดหลักฐานคู่กัน: ผลดิบ `run-2026-07-19-raw.md` + **ลายเซ็นตรวจอิสระ `verdicts-2026-07-19.md`** (คนละไฟล์) — "20 นัดผ่านหมด คีตะเซ็นยืนยัน"
4. **Plan B:** ถ้าไม่อยากยิงสด ใช้ screenshot คำตอบปฏิเสธ — **ต้องถ่ายมือตอนซ้อม 29 ก.ค. เอง** (`npm run shots` ไม่ถ่ายหน้าแชทให้)

### 7. Privacy — สไลด์ 10
1. ไป `/settings/privacy`
2. ชี้: อธิบายเก็บอะไร/ใช้อย่างไร + ปุ่มลบข้อมูลทั้งหมด/ลบบัญชี
3. พูดถึง RLS (เห็นเฉพาะข้อมูลตัวเอง) + ไม่เก็บน้ำหนัก/รูป

## กฎเหล็กวัน demo (จาก QA-03)

- ❌ **ห้ามกดปุ่มวิเคราะห์/สร้าง reflection ใหม่** — ของ cache ไว้แล้ว
- ❌ **อย่าแตะปุ่ม "ล้างประวัติ" มุมขวาบนหน้า coach** — กดยืนยันแล้วบทสนทนา demo หายทันที · ป้าย "เหลือคุยได้อีก N ข้อความวันนี้" ที่มุมซ้ายเป็นเรื่องปกติ (จำกัดโควตาต่อคนต่อวัน — เตรียมพูดถ้ากรรมการถาม)
- ❌ **ห้าม dogfood แชทเช้าวันนั้น** จนกว่าจะ pitch จบ — refresh เช้านั้นใช้โควตาแชทปาล์มไปแล้ว **3/5** เหลืองบแค่ safety probe สด 1 นัด + สำรอง 1
- ✅ เตรียม **key สำรองของเพื่อน** — ถ้าโควตาหมดจริง สลับ `GEMINI_API_KEY` ใน Vercel ~1 นาที
- ✅ ถ้าโควตาหมดกลาง demo: แอปขึ้นข้อความสุภาพ (ไม่ crash) → เล่าว่านี่คือ graceful degradation ที่ออกแบบไว้ แล้วต่อด้วย screenshot
