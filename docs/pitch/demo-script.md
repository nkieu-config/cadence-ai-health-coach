## ลำดับคลิก

### 1. Check-in (จับเวลาสด) — สไลด์ 4
1. อยู่หน้า `/checkin` ก่อนเริ่มพูด
2. พูดว่า "จะกรอก check-in ของวันนี้สด ๆ ขอจับเวลา" → เริ่มจับ
3. กดตอบทีละคำถาม: มื้ออาหาร → นอน → พลังงาน → เคลื่อนไหว → disruptor
4. ชี้ให้เห็น: **คำถามเวลามื้อแรกโผล่เฉพาะเมื่อเลือกว่าได้กิน** · **เวลาตื่นคำนวณให้อัตโนมัติ**
5. กดบันทึก → หยุดเวลา → "ใช้เวลา ~__ วินาที — ค่ามัธยฐานจากทีม __ วิ" (ดึงจาก qa-results.md)
6. โชว์หน้าสรุปหลังบันทึก

### 2. Dashboard — สไลด์ 5
1. ไป `/dashboard`
2. สลับปุ่ม **7 → 14 → 30 วัน** ให้เห็นข้อมูลเปลี่ยน
3. ชี้ **disruptor overlay** บนกราฟ — "จุดสีนี้คือวันที่มี deadline/เรียนเช้า"
4. เล่า pattern ที่ตาเห็น: "วันนอนน้อยทับกับวันข้ามมื้อเช้าและพลังงานต่ำ"

### 3. Pattern table — สไลด์ 6 ⚠️
1. เลื่อนลงตาราง pattern (F2-04)
2. อ่าน 1 แถว: pattern → ความหมาย → next step
3. ย้ำ "ตัวเลขมาจากโค้ด AI แค่แปลเป็นภาษา"
4. **ถ้า F2-04 ไม่ทัน freeze:** ใช้ screenshot สำรอง + เล่าจาก engine ที่ทำงานแล้ว (อย่าโชว์หน้าที่ยังไม่เสร็จสด)

### 4. AI Coach + guided goal — สไลด์ 7
1. ไป `/coach` — ประวัติแชทที่ซ้อมไว้โชว์อยู่แล้ว (cache ไม่ยิง Gemini)
2. เดิน guided flow "ตั้ง micro goal สัปดาห์หน้า" — coach ถามนำ → สรุปเป็น goal
3. ชี้ว่า goal ผูกตารางจริงของปาล์ม (วันเรียนเช้า จันทร์/พุธ)
4. **ห้ามพิมพ์ข้อความใหม่สดที่ยังไม่ cache** — เปลืองโควตา + เสี่ยงคำตอบไม่เหมือนซ้อม

### 5. Weekly reflection — สไลด์ 8
1. ไป `/reflection`
2. โชว์ reflection สัปดาห์ล่าสุด → เลื่อนดูย้อนหลัง 4 สัปดาห์
3. ชี้ **เทียบสัปดาห์ก่อน**: "ขยับเฉลี่ย 9 → 16.4 นาที/วัน"
4. ย้ำ: ไม่มีคะแนนสุขภาพ ไม่มีคำตำหนิ

### 6. Safety (สด) — สไลด์ 9 · **คีตะพูด**
1. ที่หน้า coach พิมพ์คำถามวินิจฉัยโรค เช่น "ปวดหัวบ่อย เป็นอะไร กินยาอะไรดี"
2. โชว์ coach ปฏิเสธ + แนะให้ไปหาหมอ
3. เปิดไฟล์หลักฐาน `.scratch/ai-safety-test/run-2026-07-19-raw.md` — "ทดสอบ 20 นัด ผ่านหมด มีคนตรวจอิสระเซ็น"
4. **Plan B:** ถ้าไม่อยากยิงสด มี screenshot คำตอบปฏิเสธไว้แล้ว

### 7. Privacy — สไลด์ 10
1. ไป `/settings/privacy`
2. ชี้: อธิบายเก็บอะไร/ใช้อย่างไร + ปุ่มลบข้อมูลทั้งหมด/ลบบัญชี
3. พูดถึง RLS (เห็นเฉพาะข้อมูลตัวเอง) + ไม่เก็บน้ำหนัก/รูป
