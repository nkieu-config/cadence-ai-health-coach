# 03 — User Persona

Prototype ใช้ persona เดียวตามข้อบังคับโจทย์ (Minimum Prototype Scope ข้อ 1)

## Primary Persona: "ปาล์ม"

| หัวข้อ | รายละเอียด |
|---|---|
| อายุ / สถานะ | 21 ปี นักศึกษาปี 3 สาขาวิศวกรรมซอฟต์แวร์ |
| ช่วงชีวิตตอนนี้ | กลางเทอม — มี group project 2 วิชา และสอบ midterm ใน 3 สัปดาห์ |
| ตารางชีวิต | เรียนเช้าบางวัน (จันทร์/พุธ 9:00), เรียน online บางวัน, ประชุมกลุ่มเย็น, โค้ดถึงดึกช่วงใกล้ deadline |
| ที่อยู่ | หอในมหาวิทยาลัย เดินถึงคณะ 10 นาที |

### พฤติกรรมปัจจุบัน

- **นอน:** ปกติ 00:30–08:00 แต่คืนก่อน deadline นอนตี 2–3 ตื่นมาพลังงานต่ำทั้งวัน
- **กิน:** วันเรียนเช้ามักข้ามมื้อเช้าเพราะตื่นไม่ทัน กินมื้อแรกตอนเที่ยง ช่วงใกล้สอบสั่งเดลิเวอรีและดื่มชานมบ่อยขึ้น
- **เคลื่อนไหว:** วันเรียน onsite ได้เดินไปคณะ วันเรียน online นั่งหน้าจอ 8+ ชม. แทบไม่ลุก

### Pain points

1. รู้ตัวว่าช่วงใกล้ deadline ชีวิตพัง แต่ไม่เห็นภาพว่าพังตรงไหนบ้างและเชื่อมกันยังไง
2. เคยโหลดแอปสุขภาพ เลิกใน 4 วันเพราะต้องกรอกเยอะ
3. อยากดูแลตัวเองแต่เป้าหมายแบบ "ออกกำลังกาย 30 นาทีทุกวัน" ทำไม่ได้จริงช่วงมี project
4. ไม่ชอบแอปที่ทำให้รู้สึกผิดเวลาทำไม่ครบ

### Goals

- มี routine ที่ดีขึ้นโดยไม่ต้องเปลี่ยนชีวิตทั้งหมด
- รู้ว่าพฤติกรรมไหนทำให้พลังงานตก
- ได้คำแนะนำที่ทำได้จริง**ระหว่าง**ทำ project ไม่ใช่คำแนะนำสำหรับชีวิตในอุดมคติ

### Scenario การใช้งาน (ใช้เป็น demo script ตอน pitch)

1. ทุกคืนก่อนนอน ปาล์มเปิดเว็บ check-in ราว 1 นาทีครึ่ง (วัดจริง QA-02: มัธยฐาน 1:26–1:30): กินกี่มื้อ นอนกี่ชม.เมื่อคืน พลังงานวันนี้ ขยับร่างกายไหม มีอะไรกระทบ routine
2. ผ่านไป 2 สัปดาห์ dashboard แสดงว่า: วันที่นอน < 6 ชม. มักเป็นวันข้ามมื้อเช้าและดื่มเครื่องดื่มหวานมากขึ้น
3. AI coach ชวนคุย: "สัปดาห์นี้วันไหนพลังงานดีสุด เกิดอะไรขึ้นวันนั้น" → ปาล์มตอบ → coach เสนอ micro goal: เตรียมขนมปัง+นมไว้ก่อนนอนสำหรับวันเรียนเช้า 2 วันหน้า
4. จบสัปดาห์ได้ weekly reflection สรุป 3 ด้าน + ข้อเสนอสัปดาห์หน้า ไม่มีคะแนน ไม่มีคำตำหนิ

## Secondary Persona (อ้างอิงใน future plan เท่านั้น ไม่อยู่ใน prototype)

**"เฟิร์น"** first jobber อายุ 23 ทำงาน hybrid หน้าจอนาน เดินทาง 1 ชม./เที่ยว — ใช้ยืนยันว่า data model รองรับ (มี `status` และ disruptor `commute`/`long_meeting` อยู่แล้ว) แต่ตัวอย่างและ seed data ทั้งหมดใช้ปาล์ม
