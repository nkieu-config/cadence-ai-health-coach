# 02 — Problem Analysis

## บริบท

นักศึกษาและ first jobber อยู่ในช่วงเปลี่ยนผ่านของชีวิต — เรียนหนักขึ้น เริ่มงานแรก ตารางไม่แน่นอน deadline ถี่ เดินทางบ่อย — ทำให้พฤติกรรมสุขภาพพื้นฐาน 3 ด้าน (กิน นอน เคลื่อนไหว) ถูกรบกวนง่าย และเสื่อมลงโดยไม่รู้ตัว

## ปัญหาหลัก แยกตามราก

> ตารางด้านล่างเป็น**ข้อสังเกตของทีมจากประสบการณ์ตรง + dogfooding** (ADR-0004) ไม่ใช่ผลวิจัยที่มี citation — ทีมงบ 0 ไม่ได้ทำ field study อย่างเป็นทางการ · ความเชื่อมโยง "นอนน้อย → ข้ามมื้อเช้า → พลังงานต่ำ" เป็น**สมมติฐานที่ระบบตั้งใจไปพิสูจน์**ด้วยข้อมูล check-in จริง ไม่ใช่ข้อสรุปสำเร็จรูป

### กลุ่มที่ 1: พฤติกรรมถูกรบกวนโดยตารางชีวิต (ปัญหาที่ระบบต้องช่วย "มองเห็น")

| ปัญหา | สาเหตุเชิงบริบท | ผลต่อเนื่อง |
|---|---|---|
| นอนดึก | project deadline, อ่านสอบ, หน้าจอ | ตื่นไม่ไหว พลังงานเช้าต่ำ |
| ข้ามมื้อเช้า / กินไม่เป็นเวลา | เรียน/งานเช้า, ประชุมยาว, เดินทาง | พลังงานตกช่วงสาย กินตามสะดวก |
| นั่งนาน เคลื่อนไหวน้อย | เรียน/ทำงานหน้าจอต่อเนื่อง | เหนื่อยล้าสะสม |

จุดสำคัญ: ปัญหาเหล่านี้**สัมพันธ์กัน** (นอนน้อย → ข้ามมื้อเช้า → พลังงานต่ำ → เคลื่อนไหวน้อย) แต่ผู้ใช้มองไม่เห็นเพราะเกิดกระจายตัวรายวัน — นี่คือช่องที่ AI pattern analysis เข้ามาตอบ

### กลุ่มที่ 2: ความล้มเหลวของวิธีแก้เดิม (ปัญหาที่ product design ต้องหลบ)

| วิธีเดิม | ทำไมล้มเหลวกับกลุ่มเป้าหมาย | Design ที่ตอบ |
|---|---|---|
| แอปสุขภาพทั่วไป | ต้องกรอกข้อมูลเยอะ ทำไม่ต่อเนื่อง | Check-in ≤ 3 นาที (Low-Burden) |
| คำแนะนำกว้าง ๆ "กินดี นอนพอ ออกกำลังกาย" | ไม่สอดคล้องชีวิตจริงที่มี deadline | ผูกคำแนะนำกับ disruptor และตารางจริง |
| เป้าหมายใหญ่ (ลดน้ำหนัก, วิ่งทุกวัน) | ล้มเหลวง่าย รู้สึกผิด แล้วเลิก | Micro goal เล็ก ล้มเหลวยาก ไม่แตะรูปร่าง |
| ระบบให้คะแนน/ตัดสิน | กดดัน รู้สึกถูกตำหนิ | ภาษาไม่ตัดสิน + weekly reflection เชิงบวก |

## คำถามที่ระบบต้องตอบให้ได้ (จาก Mission Goal)

1. ผู้ใช้มี pattern กิน–นอน–เคลื่อนไหวอย่างไร → **Dashboard + Pattern Analysis**
2. ตารางเรียน/งาน/deadline ส่งผลต่อสุขภาพอย่างไร → **Disruptor ใน check-in เชื่อมกับทุก pattern**
3. จุดแข็งที่ควรรักษา / จุดที่ควรปรับเล็ก ๆ → **Weekly Reflection**
4. AI แนะนำ routine ที่เหมาะกับบริบทอย่างไร → **Coach Conversation + Micro Goal**
5. ให้คำแนะนำปลอดภัย ไม่เกินขอบเขตการแพทย์อย่างไร → **Guardrail (docs/08)**
6. ปกป้องข้อมูลสุขภาพอย่างไร → **Privacy design (docs/08)**

## สมมติฐานหลักของ product

> ถ้าการบันทึกเบาพอ (≤ 3 นาที/วัน) และคำแนะนำเล็กพอ + ผูกกับตารางชีวิตจริง ผู้ใช้กลุ่มนี้จะบันทึกต่อเนื่องพอที่ AI จะเห็น pattern และเสนอ next step ที่เขาทำจริง

สมมติฐานนี้ทดสอบด้วย dogfooding ของทีมเอง (ADR-0004) — ถ้าทีมเองยังบันทึกไม่ต่อเนื่อง แปลว่า check-in ยังเบาไม่พอ ต้องตัดคำถามลง
