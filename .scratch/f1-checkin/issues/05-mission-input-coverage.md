## Comments

2026-07-14 (A): **มาจากการ audit แผน+เอกสารทั้งระบบเทียบ mission brief**

ตรวจโจทย์ข้อ 5 (Required Input) ทีละบรรทัดเทียบ schema จริง เจอ **4 ช่องที่ขาด** ไม่ใช่ช่องเดียว:

| โจทย์ขอ | สถานะก่อนแก้ |
|---|---|
| 5.4 ความรู้สึกหลังทำกิจกรรม | ❌ FR-1.4 เขียนว่า "หลังมื้ออาหาร**/หลังเคลื่อนไหว**" แต่ schema มีแค่ `meal_feeling` |
| 5.2 เวลากินอาหารโดยประมาณ | ❌ ไม่มีเลย |
| 5.2 ประเภทอาหาร (4 อย่าง) | ⚠️ มีแค่ 2/4 (มื้อหลัก + เครื่องดื่มหวาน) ขาดของว่าง + ผัก/ผลไม้ |
| 5.1 ช่วงที่มี deadline | ❌ FR-0.2 เขียนไว้ชัด แต่ `profiles` ไม่มีคอลัมน์ |

**ทำไมต้องทำตอนนี้ ไม่ใช่หลัง Sprint 2:** seed (INFRA-06) ยังไม่เขียน และ 3 สายเพิ่งเริ่ม → เพิ่มคอลัมน์ตอนนี้ = ไม่มีใครต้องแก้งาน · ถ้ารอ = migration กลางทางที่คนกำลังวิ่ง + seed ต้องเขียนใหม่

**เจตนา 3 อย่างที่กันไม่ให้ check-in ยาวขึ้น** (Low Burden = 1 ใน 9 เกณฑ์ให้คะแนน):

1. บริบทชีวิต (5.1) ถามครั้งเดียวตอน onboarding → ภาระรายวัน = 0
2. คำถามเสริมโผล่เฉพาะเมื่อเกี่ยว — ไม่ได้กิน ไม่ถามเวลามื้อแรก · ไม่ได้ขยับ ไม่ถามความรู้สึกหลังขยับ · **บังคับใน `validate.ts` ไม่ใช่แค่ซ่อนใน UI** (ยิง API ตรงก็สร้างแถวขัดแย้งไม่ได้)
3. **เวลาตื่นนอน: ถาม 2 ใน 3 พอ** — โจทย์ขอเวลาเข้านอน + เวลาตื่น + ชั่วโมงนอน แต่ 3 ค่านี้อิสระกันแค่ 2 ค่า · เรารับเวลาเข้านอน + ชั่วโมงนอน แล้ว**คำนวณเวลาตื่นแสดงให้ผู้ใช้เห็น** (เข้านอน 00:00–01:00 นอน 7 ชม. → ตื่นราว 07:00–08:00) · ถามข้อที่ 3 = เพิ่มภาระโดยไม่ได้ข้อมูลใหม่

**ของแถมที่ได้มาฟรี:** พอมี `first_meal_time` แล้ว pattern "กินเป็นเวลา × พลังงาน" (โจทย์ F3 ข้อ ง) ทำได้จริงเป็นครั้งแรก — ของเดิมแยกที่ "กินครบมื้อ" ซึ่งเป็นคนละคำถาม

**บั๊กที่เจอระหว่างทาง:** `completeOnboarding()` ไม่ validate อะไรเลย รับ string อะไรก็ได้ลง DB (คลาสเดียวกับบั๊กที่แก้ไปแล้วฝั่ง check-in) → เพิ่ม allow-list ใน `lib/onboarding/types.ts`
