# F5-03: goal AI ต้องกินข้อจำกัด + ตารางชีวิตจากโปรไฟล์

Status: done

Priority: M — ห้ามตัด (เกณฑ์ Personalization)
Refs: FR-5.1, F5-01, docs/00 §11 (Evaluation Criteria)

## ปัญหา

เกณฑ์ Personalization ในโจทย์เขียนว่า *"เชื่อมโยงกับ**ตารางชีวิต**และ**ข้อจำกัด**ของผู้ใช้"* และงานข้อแรกของ F5-01 ก็สั่งว่า *"ปรับตาม check-in + disruptor + **ข้อจำกัดจริงของผู้ใช้**"* — ส่วนข้อจำกัดยังไม่ได้ทำ

`buildGoalPrompt(situations, checkins)` ส่งให้ Gemini แค่สถานการณ์ + ตัวอย่าง check-in · **ไม่มีอะไรจากโปรไฟล์เลย** ทั้งที่ onboarding เก็บมาครบแล้วและ `getProfile()` ก็พร้อมใช้:

- `typical_constraints` — ไม่ค่อยมีเวลา / ไม่มีสถานที่ออกกำลังกาย / งบจำกัด / พักผ่อนไม่พอ / เดินทางนาน
- `busy_periods` — ช่วงสอบ / ส่งโปรเจกต์ / สิ้นเดือน / ทุกสัปดาห์ / ไม่แน่นอน
- `early_days` — วันที่ต้องตื่นเช้า

ผลคือความไม่สมมาตรที่อธิบายยากบนเวที: **โค้ชแชทรู้ข้อจำกัด** (`chat/context.ts` ส่งให้ครบ) แต่ **ระบบแนะนำเป้าหมายไม่รู้** — ถ้ากรรมการถามว่า "ผู้ใช้บอกว่าไม่มีสถานที่ออกกำลังกาย goal จะต่างไหม" ตอนนี้คำตอบคือไม่ต่าง

หลักฐานประกอบจาก QA-01 รอบ 16 ก.ค. เคส 09: goal ที่ได้คือ `"พักยืดเหยียด 5 นาที"` — ปลอดภัยแต่ **ยังไม่ผูกกับตารางของใครเลย**

## งาน

- [ ] `buildGoalPrompt` รับโปรไฟล์เพิ่ม แล้วใส่ข้อจำกัด/ช่วงงานหนัก/วันตื่นเช้าเป็นภาษาไทยลง prompt
- [ ] สั่งชัดใน prompt ว่า goal ต้องทำได้จริงใต้ข้อจำกัด — ห้ามเสนอสิ่งที่ขัดกัน (ไม่มีสถานที่ → ห้ามชวนไปฟิตเนส)
- [ ] `recommendGoals()` ดึง `getProfile()` มาส่งต่อ — **signature เดิมห้ามเปลี่ยน** (🟨 กำลังทำ F5-02 อยู่บน signature นี้)

- [ ] `scripts/test-goal-ai.ts` ส่งโปรไฟล์จริงของปาล์มด้วย จะได้พิสูจน์เส้นเดียวกับ production

**ทำอะไร:** `buildGoalPrompt` รับโปรไฟล์เป็นอาร์กิวเมนต์ที่ 3 (บังคับ ไม่ใช่ optional — จะได้ลืมส่งไม่ได้) แล้วใส่บล็อกตารางชีวิต+ข้อจำกัดเป็นภาษาไทยผ่าน label เดิมจาก `onboarding/types` (ท่าเดียวกับที่ `chat/context-format` ใช้อยู่) พร้อมคำสั่งห้ามเสนอสิ่งที่ขัดกับข้อจำกัด · `recommendGoals()` ดึง `getProfile()` คู่ขนานกับ `getCheckins()` · **signature สาธารณะไม่ขยับ — 🟨 ทำ F5-02 ต่อได้ไม่ต้องแก้อะไร**

**พิสูจน์ด้วย Gemini จริง 2 ครั้ง — check-in ชุดเดียวกัน เปลี่ยนแค่โปรไฟล์:**

| โปรไฟล์ | goal ที่ได้ (สถานการณ์ "มีเรียนเช้า") |
|---|---|
| ปาล์มจริง — ตื่นเช้า **จ/พ** · ไม่ค่อยมีเวลา, พักผ่อนไม่พอ | "เตรียมขนมปังหรือผลไม้ไว้ข้างเตียงสำหรับคืนก่อนวันเรียนเช้า **(จ, พ)**" |
| อีกคน — ตื่นเช้า **อ/พฤ** · ไม่มีสถานที่, **งบจำกัด**, เดินทางนาน | "จัดเตรียมของกินง่าย ๆ เช่น **กล้วย หรือขนมปัง** สำหรับเช้า**วันอังคารและพฤหัสบดี**นี้" |

- ก่อนแก้ goal นี้เคยเป็น "…สำหรับเช้าวันที่มีเรียน **1 วันในสัปดาห์นี้**" — ลอย ๆ ไม่ผูกวันจริง
- โปรไฟล์ที่มี "งบจำกัด" AI เลือกของถูกให้เอง (กล้วย/ขนมปัง) โดยไม่ต้องสั่งเจาะจง
- goal สถานการณ์ "มีเดดไลน์" เหมือนกันทั้งสองโปรไฟล์ — สมเหตุสมผล เพราะ "พัก 5 นาที" ไม่ต้องใช้เงินหรือสถานที่ ไม่มีอะไรต้องปรับ
- หลุด validation 0/2 ทั้งสองรอบ

**unit test:** ข้อจำกัดโผล่ใน prompt · โปรไฟล์ null/ว่าง ไม่พังและไม่มีบล็อกข้อจำกัด · ค่าที่ระบบไม่รู้จักถูกตัดทิ้งไม่หลุดเข้า prompt ดิบ ๆ
