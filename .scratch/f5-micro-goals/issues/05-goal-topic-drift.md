## ปัญหา (เจอตอนยิงพิสูจน์ F5-04 ที่ค้างไว้)

ผู้ใช้เลือกด้าน **"การเคลื่อนไหว"** ใน guided flow · ส่ง `constraints: [ไม่มีสถานที่, งบจำกัด]` · `busyDaysNextWeek: [ศ, ส]`

```text
situation ที่เลือก: deadline, long_screen        ← ถูกต้อง (F5-04 ทำงาน)

  มีเดดไลน์: เตรียมขนมปัง/ผลไม้ไว้ข้างเตียงคืนก่อนวันมีเดดไลน์ในวันจันทร์/พุธ   ← ❌
  นั่งหน้าจอนาน: ยืดเหยียดเบาๆ 5 นาที หลังคาบเรียน/ประชุม ออนไลน์แต่ละครั้ง      ← ✅
```

**2 ปัญหาในบรรทัดเดียว:**

1. **เปลี่ยนเรื่อง** — goal มาตรฐานของ `deadline` คือ "ตั้งเวลาพัก 5 นาทีทุก 60–90 นาที" (เรื่องพัก/ขยับ) แต่ AI ตอบเป็นเรื่อง**มื้อเช้า** · ผู้ใช้เลือก "การเคลื่อนไหว" แล้วได้ goal เรื่องกิน = คำตอบไม่ตรงคำถาม เห็นชัดบนเวที
2. **อ้างวันผิดแถว** — ใช้ "วันจันทร์/พุธ" (= `วันที่ต้องตื่นเช้า`) ทั้งที่วันแน่นสัปดาห์หน้าที่ผู้ใช้เพิ่งเลือกคือ ศ/ส · ไม่ได้ hallucinate (จ/พ อยู่ใน prompt จริง) แต่หยิบผิดความหมาย

**ต้นเหตุอยู่ที่ `buildGoalPrompt`** — เขียนว่า *"ปรับ goal มาตรฐานด้านบนให้เข้ากับข้อมูลนี้"* แต่**ไม่เคยห้ามเปลี่ยนเรื่อง** และบอกแค่ *"ใช้วันจากตารางด้านบนเท่านั้น"* โดยไม่บอกว่าแต่ละแถวหมายถึงอะไร

## งาน

- [x] `buildGoalPrompt` บอกชัดว่า **ปรับได้แค่ถ้อยคำ/วัน/จำนวน — ห้ามเปลี่ยนเรื่องจาก `goal_มาตรฐาน` ของ situation นั้น**
- [x] อธิบายความหมายของวันแต่ละแถว (`วันที่ตารางแน่นสัปดาห์หน้า` = ใช้วางแผน · `วันที่ต้องตื่นเช้า` = อ้างได้เฉพาะ goal มื้อเช้า)
- [x] **แก้ที่ user prompt เท่านั้น ห้ามแตะ `GOAL_SYSTEM_PROMPT`** — ไม่งั้นหลักฐาน safety ต้องรันใหม่
- [x] validation คำต้องห้ามไม่ถูกแตะ

## Acceptance criteria

- ยิงจริงเคส movement + ไม่มีสถานที่ + งบจำกัด → goal ของ `deadline` ต้องยังอยู่ในเรื่องพัก/ขยับ ไม่ใช่มื้อเช้า
- เคส sleep (ที่เคยผ่าน) ต้องไม่พังตาม
- unit test เดิมทั้งหมดยังเขียว
