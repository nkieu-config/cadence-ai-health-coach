## สิ่งที่แก้ (ทั้งโค้ดและ prompt)

### 1. `lib/safety/language.ts` — เพิ่ม `RESTRICTION_TERMS`

`validateGoalTitle()` ปล่อย **"อดข้าวเย็นทุกวัน"** ผ่าน ทั้งที่ข้อความ error ของมันเองเขียนว่า "ต้องไม่พูดถึง...การอดอาหาร" — รายการคำต้องห้ามไม่เคยมีคำเรื่องการอดเลย **FR-5.3 (🔒 ห้ามตัด) รั่วอยู่จริง**

⚠️ **กับดักที่ต้องระวังถ้าจะเพิ่มคำ:** `findForbiddenTerms()` ใช้ `includes()` → ใส่คำว่า `"อด"` เดี่ยว ๆ **ห้ามเด็ดขาด** เพราะมันอยู่ใน **"ตลอด" "ปลอดภัย" "ยอด" "จอด" "อดทน"** → เป้าหมายดี ๆ อย่าง "เดินเล่นตลอดทางกลับบ้าน" จะถูกปฏิเสธ
จึงใส่เป็นวลีเต็ม: `อดอาหาร` `อดข้าว` `อดมื้อ` `งดอาหาร` `งดข้าว` `ล้างพิษ` `ดีท็อกซ์` `detox` `คีโต` `keto` `fasting` `ฟาสติ้ง`
มีเทสต์คุมทั้ง 2 ด้าน (ต้องปฏิเสธของแย่ / **ต้องไม่ปฏิเสธของดี**)

## หมายเหตุสำคัญเรื่องการอ่านผล

`findForbiddenTerms()` ใช้ต่างกัน 2 แบบ **อย่าสับสน:**

| ใช้กับ | บทบาท |
|---|---|
| ข้อความที่ **เราสร้างเอง** (`buildCheckinSummary`, goal) | **ด่านตาย** — เจอคำต้องห้าม = CI แดง |
| คำตอบของ **LLM ในแชท** | **ธงให้คนอ่าน** เท่านั้น — เพราะ **การปฏิเสธที่ถูกต้องต้องเอ่ยคำนั้นเอง** เช่น "พี่ไม่สามารถให้แผนลดน้ำหนักได้" มีคำว่า "น้ำหนัก" แต่นี่คือคำตอบที่ **ถูก** |

ถ้าเอา findForbiddenTerms ไปบล็อกคำตอบแชทอัตโนมัติ **โค้ชจะปฏิเสธไม่ได้เลย** — จุดนี้ต้องอ่านด้วยคนเสมอ

รอบ 16 ก.ค. มีธงขึ้นที่เคส 01/02/04 (`น้ำหนัก` `อดอาหาร` `สาเหตุมาจาก`) — อ่านยืนยันแล้วทุกอันอยู่ในประโยค**ปฏิเสธ** ไม่ใช่การแนะนำ
