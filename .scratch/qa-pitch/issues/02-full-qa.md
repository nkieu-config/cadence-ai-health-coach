Playwright smoke suite **30 เทสต์ / ~40 วินาที** — เปิดทุกหน้าจริง × มือถือ + เดสก์ท็อป × light + dark แล้วเช็คอัตโนมัติ:

| เช็คอะไร | จับบั๊กชนิดไหน |
|---|---|
| `<h1>` อันเดียวทุกหน้า | โครงหน้าพัง / heading ซ้อน |
| ไม่มี horizontal scroll | layout ล้นบนมือถือ |
| ทุกอย่างที่กดได้ ≥ 44px | ปุ่มเล็กเกินกดพลาด |
| **contrast ≥ 4.5:1 ทั้ง light และ dark** | **ข้อความ/กราฟหายไปตอนกลางคืน** |
| ไม่มี console error | component พังเงียบ ๆ |
| **เช็คอินกรอกครบ 4 ขั้นแล้วบันทึกได้จริง** | เส้นทางเขียนข้อมูลพัง = แอปใช้ไม่ได้ |

**ที่ QA-02 ต้องเติมเอง:** เดิน scenario ปาล์มจบเส้นบน production · จับเวลาเช็คอินจริง · เทสต์บนมือถือจริง (Playwright ใช้ chromium จำลอง — **ควรเปิด Safari บน iPhone จริงอย่างน้อย 1 รอบ**) · เทสต์ flow ของ AI (โควตา 20 ครั้ง/วัน ระวัง)

**บทเรียนจากการเขียนมัน — จดไว้เพราะจะกัดคนอื่นซ้ำ:**

1. **ฟอร์มเช็คอินเติมค่าเดิมจากบันทึกที่มีอยู่** → กดชิปที่เลือกอยู่แล้ว = **ปิดมันทิ้ง** · เทสต์ต้องเช็ค `aria-pressed` ก่อนกด ไม่งั้นผลจะเปลี่ยนไปตามว่าใครเช็คอินอะไรไว้
2. **Tailwind v4 คืนสีเป็น `oklab()` ไม่ใช่ `rgb()`** → ถ้า parse สตริงเอง จะอ่าน contrast ผิดหมด (ผมเจอ false failure 14 จุดบนหน้า privacy) · **ต้องให้เบราว์เซอร์แปลงสีให้** (วาดลง canvas แล้วอ่าน pixel)
3. **เทสต์ที่ fail ผิด ๆ อันตรายกว่าไม่มีเทสต์** — คนจะเริ่มเมินมัน
