# UX-04 — แถบวัน (day lines)

Status: ready-for-agent

## ปัญหา

โจทย์ Feature 3 บอกว่าคุณค่าของแอปคือ **"เชื่อมโยง 3 ด้านเข้ากับบริบทชีวิต"** แต่ dashboard **แยก 3 ด้านออกจากกัน** — การ์ดนอน การ์ดกิน การ์ดขยับ คนละใบวางเรียงกัน

ผู้ใช้เลยต้อง**อ่านคำบรรยายของ AI ถึงจะรู้ว่ามันเกี่ยวกัน** ทั้งที่ควรเห็นด้วยตา

## ขอบเขตที่ข้อมูลรองรับจริง (สำคัญ)

ความคิดแรกคือวางทั้ง 3 ด้านบนแกนเวลา 24 ชม. **แต่ schema ไม่มีข้อมูลนั้น:**

| ข้อมูล | มีอะไรจริง | วางบนแกนเวลาได้ไหม |
| --- | --- | --- |
| การนอน | `bedTimeBucket` (5 ช่วง) + `sleepHours` | ✅ |
| มื้อแรก | `firstMealTime` (3 ช่วง) | 🟡 หยาบ |
| มื้ออื่น | ไม่มีเวลา | ❌ |
| การขยับ | มีแต่จำนวนนาที | ❌ |

**ห้ามเดาตำแหน่งให้ดูสวย** — ผิดกฎเหล็กที่ว่าตัวเลขต้องมาจากข้อมูลจริง กราฟที่แต่งตำแหน่งเองคือกราฟที่โกหก

จึงทำเวอร์ชันที่ซื่อสัตย์: **แกนเวลาแสดงเฉพาะสิ่งที่มีเวลาจริง** (นอน + มื้อแรก) · การขยับอยู่เลนแยกเป็นความยาว ไม่อ้างเวลา

## ทำอะไร

แถวละ 1 วัน 7 วันซ้อนกัน · แกน 21:00 → 14:00 (ช่วงที่เรื่องมันเกิด: เข้านอน → ตื่น → มื้อแรก)

- แท่งนอน วางตามช่วงเวลาเข้านอนจริง ยาวตามชั่วโมงที่นอน
- จุดมื้อแรก ตามช่วงที่บันทึก
- วันที่มี disruptor แถบสีพาดทั้งแถว
- วันที่ไม่ได้บันทึก = เส้นประจาง **ไม่ใช่ 0 ไม่ใช่สีแดง**
- การขยับอยู่ขวาสุด เป็นความยาว + ตัวเลขนาที

ผลที่ควรได้: **เห็นว่าแถบเดดไลน์พาดตรงที่แท่งนอนหดสั้น แล้วจุดมื้อแรกเลื่อนไปบ่าย** — pattern ที่ AI อธิบายเป็นย่อหน้า กลายเป็นรูปทรงที่มองแวบเดียวรู้

## ที่วาง

วางเหนือ `PatternTable` — รูปหลักฐานอยู่ติดกับบทวิเคราะห์ที่อ้างถึงมัน

**ไม่ลบตาราง pattern ของแพรรี่ (F2-04)** — เป็น deliverable ข้อ 8 · แลกมาด้วยหน้ายาวขึ้นราว 1 จอ ถ้าอยากตัดต้องคุยกันก่อน
