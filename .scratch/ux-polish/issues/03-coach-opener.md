# UX-03 — โค้ชเปิดด้วยคำถามจากข้อมูลจริง

Status: ready-for-agent

## ปัญหา

หน้าโค้ชของบัญชี demo เปิดมาเจอ**กล่องว่างสูง ~700px** มีไอคอนลูกโป่งกับข้อความชวนคุยกลาง ๆ ส่วนคำถามแนะนำ 2 อันตกอยู่ใต้ fold

สองข้อเสีย:

1. **ช่องแชทเปล่า + จุดกระพริบ = ภาพจำของ AI ทั่วโลก** ดูเหมือน chatbot ที่ใครก็ทำได้ ไม่เหมือนโค้ชที่รู้จักผู้ใช้
2. **โยนภาระคิดคำถามให้ผู้ใช้** ทั้งที่โจทย์ Feature 4 บอกว่าหัวใจคือ**คำถามที่โค้ชถาม** (มีตัวอย่าง 5 ข้อในโจทย์ ทุกข้อเป็นคำถามที่โค้ชเป็นคนเริ่ม)

## ทำอะไร

ให้ server คำนวณ "ประเด็นเปิด" จากบันทึกจริง 7 วันล่าสุด แล้วให้โค้ชทักก่อน

```
วันพุธที่ 16 ก.ค. เป็นวันที่พลังงานต่ำที่สุดในสัปดาห์
วันนั้นมีเดดไลน์ด้วย — เกิดอะไรขึ้นบ้าง
```

- ตัวเลข/วันมาจาก**บันทึกจริง ไม่ใช่ให้ LLM คิด** (กฎเหล็ก: สถิติมาจากโค้ด)
- **ไม่ยิง Gemini ตอนเปิดหน้า** — เป็นข้อความที่ประกอบจาก template ในโค้ด ไม่กินโควตา ไม่มี latency
- ข้อมูลไม่พอ (< 3 วัน) → ถอยกลับไปใช้ข้อความชวนคุยแบบเดิม

## ทำไมได้คะแนน

| เกณฑ์ | ได้ยังไง |
| --- | --- |
| AI Usefulness 15% | โค้ชอ้างข้อมูลของผู้ใช้ได้ตั้งแต่ประโยคแรก ไม่ต้องรอผู้ใช้ป้อน |
| Personalization 12% | คำถามผูกกับวันจริง + disruptor จริงของคนคนนั้น |
| Prototype Quality 15% | วัน pitch เปิดหน้าโค้ชแล้ว**มีอะไรให้ดูทันที** ไม่ใช่กล่องเปล่า |
