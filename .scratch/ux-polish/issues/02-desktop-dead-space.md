# UX-02 — อุดช่องว่างตายบนเดสก์ท็อป

Status: ready-for-agent

## ทำไมสำคัญ

**วัน pitch ฉายขึ้นโปรเจกเตอร์ = กรรมการเห็นเดสก์ท็อป ไม่ใช่มือถือ** แต่ทั้งโปรเจกต์ออกแบบ mobile-first · จอที่ขัดน้อยที่สุดคือจอที่กรรมการเห็น

layout เดสก์ท็อปไม่ได้ผิด (sidebar / step rail / ตาราง-การ์ด ทำงานดี) ปัญหาคือ**ที่ว่างที่ไม่มีอะไรอยู่** ซึ่งบนจอใหญ่จะเห็นชัดมาก

## จุดที่เจอ (จากภาพจริง 20 ก.ค.)

| จุด | อาการ |
| --- | --- |
| การ์ด "สรุปวันนี้" | grid ยืดให้สูงเท่ากราฟข้าง ๆ + `justify-between` ดันปุ่มลงล่าง → ว่างกลางการ์ด ~200px |
| การ์ด "สรุปสัปดาห์" | มีแค่บรรทัดเดียว + ปุ่ม แต่กินครึ่งแถว ทั้งที่ `Reflection` มี `strengths` / `nextWeek` ให้ใช้อยู่แล้ว |
| Step rail หน้าเช็คอิน | 4 ขั้นอยู่บน แล้วว่างยาวถึงท้ายจอ |

> ช่องว่างในการ์ดดูเหมือน**บั๊ก** · การ์ดสูงไม่เท่ากันดูเหมือน**ตั้งใจ** — เลือกอย่างหลัง

## ทำอะไร

1. `lg:items-start` ที่ grid ของ dashboard + ถอด `h-full` / `justify-between` ออกจาก `TodaySummary` → การ์ดสูงตามเนื้อหา
2. `ReflectionCard` แสดง `strengths` จริงแทนที่จะมีแต่ปุ่ม — เป็นเนื้อหาที่ AI สร้างไว้แล้ว ไม่ต้องเรียกเพิ่ม
3. `StepRail` เติม "ดูบันทึกย้อนหลัง" + ข้อความบอกว่ากรอกไม่จบก็กลับมาต่อได้ (ตอบโจทย์ low-burden ตรง ๆ)

## ไม่ทำ

- ไม่รื้อ grid / breakpoint / PageContainer — ระบบเดิมถูกแล้ว
- ไม่แตะ sidebar (ต้องดึงข้อมูลใน layout = เสี่ยงเกินกว่าที่ได้)

## เสร็จเมื่อ

- [ ] `npm run e2e` เขียว
- [ ] ไม่มีช่องว่างเกิน ~1 บรรทัดในการ์ดใดบนเดสก์ท็อป
