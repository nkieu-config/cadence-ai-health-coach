# INFRA-07: โควตา Gemini ฟรี 20 ครั้ง/วัน — กันแอปตายวัน pitch

Status: ready-for-agent
Owner: A
Sprint: 2
Priority: M — ถ้าไม่แก้ demo วัน pitch มีสิทธิ์ล่มกลางเวที
Refs: ADR-0003, NFR-2, docs/09 Risk Register
Blocked by: —

## ปัญหา (วัดจริง ไม่ใช่เดา)

ตอนรัน F3-02 ยิงชนเพดานโควตาจริง — ตัวเลขนี้มาจาก error ของ Gemini API:

```json
"quotaId": "GenerateRequestsPerDayPerProjectPerModel-FreeTier"
"quotaValue": "20"
"model": "gemini-2.5-flash"
```

| โมเดล | โควตา free tier (วัด 14 ก.ค. 2026) |
|---|---|
| `gemini-2.5-flash` ← ที่เราใช้ | **20 ครั้ง/วัน/โปรเจกต์** |
| `gemini-2.5-flash-lite` | HTTP 404 — ไม่เปิดให้ผู้ใช้ใหม่แล้ว |
| `gemini-2.0-flash-lite` | `limit: 0` |

**ADR-0003 กำหนดว่า production ใช้ API key เดียวร่วมกันทั้งแอป**
→ ทั้งระบบบน production มีโควตารวม **20 ครั้ง/วัน**

- แชทโค้ช 1 ข้อความ = 1 ครั้ง
- ซ้อม pitch 2 รอบ × โชว์แชท 5 ข้อความ = 10 ครั้ง → **เหลือ 10 ครั้งสำหรับวันจริง**
- ถ้ากรรมการขอลองพิมพ์เอง หรือทีมเผลอ dogfood วันนั้น → **หมดกลางเวที**

## งาน

- [ ] **Cache ต้องกันการยิงซ้ำจริง** — `getLatestInsight()` / `getLatestReflection()` ต้องอ่าน `ai_outputs` ก่อนเสมอ · ปุ่ม "สร้างใหม่" ต้องไม่ยิงถ้ามีของช่วงเดิมอยู่แล้ว (ตรวจว่าทำแบบนี้จริงใน F3-03 / F6-01)
- [ ] **จำกัดจำนวนข้อความแชทต่อผู้ใช้ต่อวัน** ใน `sendCoachMessage()` — กันคนเดียวใช้โควตาหมดทั้งทีม
- [ ] **ข้อความตอนโควตาหมดต้องไม่ใช่ error หน้าขาว** — ต้องเป็น "โค้ชขอพักแป๊บนะ ลองใหม่พรุ่งนี้" + ปุ่ม retry (NFR-2)
- [ ] **แผนสำรองวัน pitch** — จดใน `qa-pitch/03-pitch-deck.md`:
  - เตรียม key สำรอง (ของเพื่อนในทีม) ไว้สลับใน Vercel env ได้ใน 1 นาที
  - **generate insight + reflection ของ demo account ล่วงหน้า 1 วัน** → วัน pitch อ่าน cache ไม่ยิง API เลย
  - screenshot สำรองทุกหน้า AI (มีในแผนอยู่แล้ว — ทำจริง)
- [ ] อัปเดต ADR-0003 ด้วยตัวเลขจริง + Risk Register ใน docs/09

## Acceptance criteria

- เปิดหน้า dashboard / reflection ซ้ำ 10 ครั้ง → **ยิง Gemini 0 ครั้ง** (อ่าน cache ล้วน)
- โควตาหมด → ผู้ใช้เห็นข้อความเป็นมิตร + ปุ่มลองใหม่ ไม่ใช่หน้า error
- ซ้อม pitch เต็มรูปแบบได้โดยไม่แตะโควตาของวันจริง

## Comments

2026-07-14 (A): **เจอตอนรัน safety checklist ของ F3-02 — ไม่ได้ตั้งใจหา แต่เจอเพราะยิงจริง**

Risk Register ใน docs/09 เขียนไว้ตั้งแต่แรกว่า "Gemini rate limit/ล่ม ตอน demo — โอกาส กลาง / ผลกระทบ สูง" แต่**ไม่มีใครรู้ตัวเลขจริง** ตอนนี้รู้แล้ว: **20**
