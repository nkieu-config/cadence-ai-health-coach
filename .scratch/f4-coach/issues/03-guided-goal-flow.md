## กำแพง: flow ถามคำถาม แต่ยังไม่มีทางส่งคำตอบเข้าเครื่องยนต์

งานข้อแรกเขียนว่า coach ถาม 3 คำถาม (เริ่มจากกิน/นอน/ขยับ · สัปดาห์หน้าวันไหนแน่น · **ข้อจำกัดอะไร**) → เสนอ goal 2 ตัวเลือก · แต่ของที่มีให้เรียกตอนนี้:

- `recommendGoals()` — **ไม่รับอาร์กิวเมนต์เลย** ปั้น goal จาก check-in + โปรไฟล์ onboarding เท่านั้น
- ใน `lib/goals` ไม่มีอะไรรับบทสนทนา · และกติกาข้อ 5 ห้ามคุณเรียก Gemini เอง

แปลว่าถ้าผู้ใช้ตอบในแชทว่า *"สัปดาห์หน้าไม่มีเวลาเลย"* goal ที่เสนอจะไม่รู้เรื่องนั้น — **นี่คือ demo หลักที่กรรมการนั่งดู ถามแล้วคำตอบไม่มีผล เขาจับได้ทันที**

## ทำยังไง — เริ่มได้เลย ไม่ต้องรอ A

**ใช้ `recommendGoals()` ตัวปัจจุบันไปก่อน** แล้วทำ UI ให้จบ:

- `recommendGoals()` → `{ ok: true, suggestions: [{ title, situation }] }` หรือ `{ error }` · คืนสูงสุด 2 ข้อ **ผ่าน validation คำต้องห้ามมาแล้วทุกอัน**
- ผู้ใช้เลือก/แก้ → `acceptGoal(title, situation)` — `@/lib/goals/actions` · validate ซ้ำให้อัตโนมัติ ไม่ต้องเรียกอะไรเพิ่ม
- ระหว่าง flow ถ้าอยากให้โค้ชตอบเป็นข้อความ ใช้ `sendCoachMessage()` เหมือน F4-01 — โค้ชเห็นโปรไฟล์+ข้อจำกัด+บันทึกจริงอยู่แล้วผ่าน `buildCoachContext`

**อย่าไปนั่งหาทางส่งคำตอบเข้า AI เอง** — A จะเพิ่ม `recommendGoals(context?)` (พารามิเตอร์เสริม ของเดิมไม่พัง) ให้รับคำตอบจาก flow ไปต่อท้าย prompt · พอ merge คุณแค่ส่ง string เข้าไป ไม่ต้องรื้อ UI · **ติดต่อ A ก่อนเริ่มส่วนนี้ อย่าเพิ่งทำเอง**

## เรื่องที่ต้องรู้ในโซนคุณ

- **คำตอบโค้ชมี markdown ดิบโผล่** (`**ข้อความ**`) เพราะฟองข้อความเรนเดอร์เป็น plain text — **A รับไปแก้เอง** อย่าไปแตะ `message-variants.tsx` จะได้ไม่ชนกัน

- **เทสด้วยบัญชีสมัครทิ้ง อย่าใช้ปาล์ม** — ปาล์มคือบัญชี demo วัน pitch ข้อความเทสที่ค้างจะโผล่บนเวที (กติกาใหม่ในบอร์ด)
